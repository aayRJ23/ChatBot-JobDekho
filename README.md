## Overview

The chatbot is a **standalone Flask microservice** that answers user questions about the Job Dekho portal. It uses a **machine learning pipeline** (TF-IDF + Logistic Regression) trained on a curated question-answer dataset (`JobPortalDataset.csv`).

The React frontend renders a floating **💬 chat widget** that sends user queries to this service and displays responses inline — no external AI API needed, no cost, fully offline-capable.

---

## How It Works

```
User types a question
        ↓
React Chatbot widget (Chatbot.jsx)
        ↓  POST /predict  { "User Input": "..." }
Flask API (app.py)  →  Loads job_portal_model.pkl
        ↓
TF-IDF Vectorizer converts text to features
        ↓
Logistic Regression classifier predicts the closest response
        ↓
Returns { "Prediction": "..." }  →  Displayed in chat bubble
```

The model is a **scikit-learn Pipeline**:
1. **TfidfVectorizer** — converts raw text to TF-IDF feature vectors (unigrams + bigrams)
2. **LogisticRegression** — multi-class classifier that matches input to a known response

## Environment Variables (.env Setup)

The Flask chatbot **does not use a `.env` file by default** — it has no database or external service credentials. The only configurable value is the port, which defaults to `5000`.

If you want to change the port or enable production mode, you can set these shell environment variables before running:

```bash
# Optional: change the port (default is 5000)
export FLASK_PORT=5000

# Optional: disable debug mode in production
export FLASK_DEBUG=0
```

Or pass them inline:
```bash
FLASK_DEBUG=0 python app.py
```

### Frontend `.env` (in the Client repo)

The React frontend needs to know the chatbot's URL. In `Project-Job-Dekho-Client/.env`:

```env
VITE_CHATBOT_URL=http://127.0.0.1:5000
```

Change this to your deployed chatbot URL when deploying to production.

## Installation & Running

### Prerequisites

- Python 3.9+
- pip

### Steps

```bash
# 1. Clone the repo
git clone https://github.com/aayRJ23/ChatBot-JobDekho.git
cd ChatBot-JobDekho

# 2. (Recommended) Create a virtual environment
python -m venv venv

# On Windows:
venv\Scripts\activate
# On Mac/Linux:
source venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Start the Flask server
python app.py
```

The chatbot API will be available at **http://127.0.0.1:5000**

> The pre-trained `job_portal_model.pkl` is already included in the repo — you don't need to retrain unless you change the dataset.

## Training the Model

If you update `JobPortalDataset.csv` with new question-answer pairs, re-run the training script to generate a fresh model:

```bash
python train_model.py
```

This will:
1. Load `JobPortalDataset.csv`
2. Split data into training (90%) and test (10%) sets
3. Train a TF-IDF + Logistic Regression pipeline
4. Save the model to `job_portal_model.pkl`
5. Write accuracy and a classification report to `training_report.txt`

Example output:
```
Loading dataset...
  Rows loaded     : 90
  Unique responses: 75
  Training samples: 81
  Test samples    : 9

Training model...
  Accuracy: 88.89%

Model saved   → job_portal_model.pkl
Report saved  → training_report.txt
```

> **Tip:** Accuracy improves significantly when you add more varied phrasings (paraphrases) for each question in the dataset.

## API Reference

### `GET /`
Returns the standalone HTML test page. Useful for quick manual testing in a browser without the React frontend running.

---

### `POST /predict`

Accepts a user question and returns the chatbot's predicted response.

**Request**
```json
Content-Type: application/json

{
  "User Input": "How do I apply for a job?"
}
```

**Response — Success (200)**
```json
{
  "Prediction": "Open the job listing you want and click the Apply Now button. Fill in your name, email, phone number, address, cover letter and upload your resume then click Send Application."
}
```

**Response — Error (400)**
```json
{
  "error": "Missing \"User Input\" key in request body"
}
```

**Response — Error (500)**
```json
{
  "error": "<exception message>"
}
```

## Dataset Format

`JobPortalDataset.csv` contains two columns:

| Column | Description |
|--------|-------------|
| `User Input` | A question or phrase the user might type |
| `Response` | The exact answer the bot should give |

Example rows:
```csv
User Input,Response
How do I create an account?,"Go to the Register page and fill in your name, email, phone number, password and select your role as either Job Seeker or Employer then click Register."
How to register on Job Dekho?,"Go to the Register page and fill in your name, email, phone number, password and select your role as either Job Seeker or Employer then click Register."
How do I apply for a job?,"Open the job listing you want and click the Apply Now button. Fill in your name, email, phone number, address, cover letter and upload your resume then click Send Application."
How do I post a job?,"You must be logged in as an Employer. Click POST NEW JOB in the sidebar, fill in the title, category, country, city, location, salary type, salary amount and description then click Create Job."
```

**Best Practice:** Add **3–5 different phrasings** for each answer. The more variety in "User Input", the better the model generalises to real user queries.

## Extending the Chatbot

To add new Q&A pairs:

1. Open `JobPortalDataset.csv`
2. Add rows with new `User Input` phrasings and their `Response` values
3. Retrain the model:
   ```bash
   python train_model.py
   ```
4. Restart the Flask server:
   ```bash
   python app.py
   ```

To test from the terminal without starting the React app:

```bash
curl -X POST http://127.0.0.1:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"User Input": "How do I post a job?"}'
```

## Notes

- The model is a **retrieval-style** classifier — it picks from a fixed set of known responses. It does not generate new text. This makes it deterministic, fast, and easy to control.
- For better accuracy at scale, consider switching to a **sentence-transformer** embedding model (e.g. `sentence-transformers/all-MiniLM-L6-v2`) with cosine similarity for matching.
- CORS is enabled for all origins (`*`) in development. For production, restrict to your frontend domain in `app.py`.
